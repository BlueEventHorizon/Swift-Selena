# タスク実行開始コマンド

## オプション

| オプション | 説明 |
|-----------|------|
| (なし) | 1タスクのみ実行して停止 |
| `--continue` または `-c` | タスクを連続実行（エラーまたは全タスク完了まで） |
| `--count N` または `-n N` | 指定した数のタスクを実行して停止 |

## **Critical Requirements:**

### 基本フロー（各タスク）
1. 計画書 `project/{feature}/plan/{feature}_plan.md` からタスクを選択
2. 必要な要件定義書・設計書を確認（project-advisor Subagent活用）
3. 必要なルール文書を確認（docs-advisor Subagent活用）
4. タスクを実行
5. ビルド・テスト確認
6. 完了報告

### 実行モード判定

**引数**: $ARGUMENTS

- 引数なし → 1タスクのみ実行して停止
- `--continue` または `-c` → タスク完了後、自動的に次のタスクを選択・実行（停止条件まで継続）
- `--count N` または `-n N` → N個のタスクを実行して停止

### 停止条件（--continue時）
- 全タスク完了
- ビルドエラー発生
- 依存関係の問題で次タスクが実行不可
- 人間の判断が必要な問題発生

### 連続実行時の注意
- 各タスク完了時に簡潔な完了報告を出力
- エラー発生時は詳細を報告して停止
- 次タスクの選択理由を簡潔に表示
