# AI対話・開発ルール

## 1. 言語ルール

### 1.1 対話言語
- **生成AIとの対話は日本語で実施**
- **技術用語・英単語はそのまま使用可能**

### 1.2 コメント記述
- **ソースコードのコメントは必ず日本語で記述**

## 2. ドキュメント作成ルール

### 2.1 スタイルガイド
- **アイコンは基本使用しない**

### 2.2 ファイルヘッダー [MANDATORY]
- 新規作成ファイルの`Created by`は生成AIではなく、下記コマンドで得られるgit定義の作者を記述：
  ```bash
  git config user.name
  ```

## 3. 開発の前提条件

### 3.1 基本定義
- **Swift-Selena**: MCP Server for Swift code analysis (Swift Package)

### 3.2 開発ルール
- **不明点は、人間に確認して事前に明確化すること**

### 3.3 優先順位 [MANDATORY]
- **要件定義書(project/{feature}/spec/**/*.md)が、すべてのドキュメントに優先します。実装に関しても最優先で内容を守ること**

### 3.4 プラットフォーム要件 [MANDATORY]

**技術要件は `Package.swift` から取得すること**

| 項目 | 説明 |
|------|------|
| `platforms` | 対象プラットフォーム（macOS 13.0+） |
| `swift-tools-version` | Swiftバージョン（5.9） |

### 3.5 プロジェクト構成

**Swift Package構成**: `Sources/`配下にソースコード、`Tests/`配下にテストコード。

## 4. 開発の役割分担

### 人間の役割

- **意思決定者**
- 要件定義、設計書、計画書、コードの作成・変更を承認

### 生成AIの役割

- 要件定義、設計書、計画書、コードの作成・変更
- 計画書の管理（タスク進捗情報の更新）

## 5. 開発時の基本原則

### 1. 仕様を必ず最初に確認する

新しいツール、ライブラリ、記法を使う前に公式ドキュメントを読んでください。

- 思い込みで実装しない
- 「知っている」と思っても確認する
- バージョンの仕様変更を把握する

### 2. 小さく試してから大きく展開する

一度に大量生産せず、1つを完成させてから次へ進んでください。

- 1つのサンプルで動作確認
- 問題なければ他に展開
- エラーが出てから修正では遅い

### 3. 品質を最優先する

スピードより正確性を重視してください。

- 推測より確認
- 複雑な正規表現を使って一括置換や、一括変更しない
  - 間違った場合に手戻りが膨大です
  - 単純置換などはOK
- 手抜きしない（全力で作業に向き合ってください）
