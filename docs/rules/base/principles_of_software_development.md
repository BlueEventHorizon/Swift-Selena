# ソフトウェア開発の原則

以下の原則を厳守すること。

## SOLID原則

- **単一責任の原則（SRP）**
- **開放閉鎖の原則（OCP）**
- **リスコフの置換原則（LSP）**
- **インターフェース分離の原則（ISP）**
- **依存性逆転の原則（DIP）**

## その他の重要な原則

- **DRY（Don't Repeat Yourself）** - 同じロジックの重複を避ける
- **KISS（Keep It Simple, Stupid）** - 不必要な複雑さを避ける
- **YAGNI（You Aren't Gonna Need It）** - 現在必要のない機能は実装しない
- **関心の分離** - 異なる機能や責任を明確に分離
- **高凝集・疎結合** - 関連機能をまとめ、依存を最小化
- **防御的プログラミング** - 適切なエラーハンドリング
- **可読性の重視** - 理解しやすいコードを書く
